# Mini-Polish Weekend — Journal

> Running log for the Mini-Polish weekend (visual foundation: noir palette, typography, small UX polish). Filled in across the weekend; not a retrospective written after the fact.

## What I built

- Step 1 — Spec + ADR + journal stub. Committed `docs/specs/mini-polish-visual-foundation.md` as the contract for the weekend, recorded the tone/palette/typography decision as `docs/decisions/0011-noir-palette-and-typography.md`, and seeded this journal file. No code changes in this step — Step 2 onward touches theme, fonts, and components.
- Step 2 — Noir palette + dark-only. Replaced shadcn's default light/dark OKLCH variables in `src/app/globals.css` with the noir palette per ADR-0011 (background `#0F0E0C`, surface `#1A1816`, border `#2B2724`, foreground `#E8DFCF`, muted-foreground `#9A8E7A`, primary/accent `#C19A4F`, destructive `#A94442`). Duplicated the same values into `:root` and `.dark` so the page stays correct even if the dark class is ever stripped. Hardcoded `class="dark"` on `<html>` in `src/app/layout.tsx` (no `next-themes`, no toggle). Left chart and sidebar tokens defined for shadcn compatibility but pointed them at our palette since nothing in the game uses them. No hardcoded color utilities to replace — `bg-slate-*` / `text-zinc-*` etc. did not appear in app code; only `bg-black/10` in `src/components/ui/dialog.tsx` (vendored shadcn primitive, semi-transparent overlay — left alone per ADR-0006).
- Step 3 — Three role-typed fonts. Added Crimson Text via `next/font/google` (`weight: ['400', '600', '700']`, `display: 'swap'`) bound to `--font-serif`. Renamed the existing Geist Sans / Geist Mono `next/font` variables from `--font-geist-sans` / `--font-geist-mono` to `--font-sans` / `--font-mono` so the Tailwind v4 `font-*` utilities resolve correctly through the `@theme inline` block. Updated globals.css `@theme inline` to register `--font-serif` (and re-pointed `--font-heading` at it). The existing `html { @apply font-sans }` base rule keeps Geist Sans as the body default. Applied `font-mono` to assistant chat bubbles and `font-sans` (explicit, for clarity) to user bubbles in `src/features/interrogation/chat-view/chat-view.tsx` to create the "transcript vs your questions" split. No `font-serif` applied yet — Briefing typography lands in Step 6.
- Step 4 — InitialAvatar. New `src/components/initial-avatar/{initial-avatar.tsx,utils.ts,index.ts}` per ADR-0006 folder+barrel — `computeInitials` and `hashToHue` are extracted into `utils.ts` so they can be unit-tested as pure functions. The avatar is a circular `div` with a deterministic `linear-gradient(135deg, hsl(h 35% 28%), hsl(h 25% 16%))` derived from the suspect name, three sizes (`sm` 32px / `md` 48px / `lg` 64px), `aria-hidden` so screen readers and Playwright role queries skip it, and `font-sans font-semibold text-foreground` for the initials. Replaced the three placeholder squares: `suspect-card.tsx` (md, dropped its local `initialsFor`), `suspect-tabs.tsx` (sm, restructured the tab to put avatar on the left and stack name + one-liner to the right), `suspect-picker.tsx` (md, dropped its local `initialsFor`). Added 10 new unit tests in `tests/unit/components/initial-avatar.test.ts` covering both helpers (initials edge cases, hue determinism, hue range).
- Step 5 — Chat polish. Three small chat-area improvements: (a) Empty state in `chat-view.tsx` — when the active suspect has no messages and no error, render a centered, italic, muted prompt `Ask {suspectName} your first question…` instead of leaving the panel blank; threaded `suspectName` from `interrogation-room.tsx` since `ChatView` reads messages straight from the store and didn't already know which suspect was active. (b) Animated typing indicator — replaced the Tailwind `animate-bounce` triple-dot in `typing-indicator/typing-indicator.tsx` with three `<span class="dot dot-N" />` nodes wrapped in `<div role="status" aria-label="Suspect is responding">`, plus a `@keyframes dot-pulse` rule and `.dot` / `.dot-1` / `.dot-2` / `.dot-3` selectors in `globals.css` (1.4s ease-in-out infinite, 0/150/300ms staggered delays, dot color `var(--muted-foreground)`). CSS-only, no Motion library. (c) Refreshed the chat textarea placeholder in `message-input.tsx` from "Ask the suspect a question…" to "What do you want to ask?" — friendlier, less repetitive with the new empty state. Did not touch the accusation modal placeholder (already says "What evidence convinces you it was them?") or briefing typography (Step 6).
- Step 6 — Briefing typography. Applied the §4.7 typographic block to `briefing-screen.tsx`. Title (`<h1>`) is now `font-serif text-4xl md:text-5xl font-semibold tracking-tight` — Crimson Text 600 with slightly tightened letter-spacing reads as a real title rather than a label. Premise wraps in `<div className="font-serif text-lg leading-relaxed max-w-prose">` and splits on `\n\n` into `<p className="mb-4 last:mb-0">` paragraphs (the case-01 premise was already 2 paragraphs from Weekend 2, so no content rewrite was needed). The `<h2>Suspects</h2>` heading was demoted from a competing `text-xl font-semibold` to a small-caps label (`font-sans text-sm uppercase tracking-wider text-muted-foreground mb-4`) so it reads as a section divider, not a second headline. Help line moved from below the suspect cards to between the suspects section and the Begin button — `font-sans text-sm text-muted-foreground italic mt-8 mb-6 text-center` — so it sits as a final hint right before the action. Begin button got wrapped in a `flex justify-center` container with `w-full min-w-[240px] md:w-auto` so it stretches on mobile and pins to a comfortable 240px on desktop. Outer container kept its existing `mx-auto max-w-2xl px-6 py-12`.

## What was hard

- shadcn's globals.css uses OKLCH, but the noir palette in the spec is hex. Wrote a one-off Node script (sRGB → linear → Oklab → Oklch) to convert the seven source hex values rather than eyeballing them. Kept the source hex in CSS comments so future edits can round-trip without re-deriving.
- The shadcn Nova preset shipped with a latent font bug: `next/font` bound Geist Sans to `--font-geist-sans`, but `globals.css` `@theme inline` had `--font-sans: var(--font-sans);` — a self-referential mapping. So Tailwind's `font-sans` utility was resolving to nothing and the page silently fell back to the browser default sans-serif rather than Geist. Fixed by renaming the next/font variables to `--font-sans` / `--font-mono` and registering `--font-serif` alongside, then verifying via `getComputedStyle` on synthetic `font-sans` / `font-mono` / `font-serif` nodes that all three resolve to the right family.
