CSS-only, no Motion library. (c) Refreshed the chat textarea placeholder in `message-input.tsx` from "Ask the suspect a question…" to "What do you want to ask?" — friendlier, less repetitive with the new empty state. Did not touch the accusation modal placeholder (already says "What evidence convinces you it was them?") or briefing typography (Step 6).

Begin button got wrapped in a `flex justify-center` container with `w-full min-w-[240px] md:w-auto` so it stretches on mobile and pins to a comfortable 240px on desktop. Outer container kept its existing `mx-auto max-w-2xl px-6 py-12`.
- Step 7 — Accusation submit loading state. Added a defensive `isSubmitting` local state to `accusation-modal.tsx` that gates both Submit and Cancel during submission. The handler is now `async`, sets `isSubmitting = true` before calling `evaluateAccusation` + `submitAccusation`, and only resets on a `try/catch` recovery path (success unmounts the modal). The Submit button label flips to `Submitting…` (Unicode ellipsis, U+2026) while in flight, and `disabled` adds `|| isSubmitting` so a rapid double-click can't fire the mutation twice. The modal-open reset block also sets `isSubmitting = false` so a previous error path leaves the next open clean. In practice the `Submitting…` label rarely paints because `evaluateAccusation` is synchronous and React batches the open/close in the same flush — but the pattern is now correct and ready for W5's LLM-as-judge accusation validation. No spinner icon, no animation; full async-feedback polish stays on Weekend 4.

## What was hard

- shadcn's globals.css uses OKLCH, but the noir palette in the spec is hex. Wrote a one-off Node script (sRGB → linear → Oklab → Oklch) to convert the seven source hex values rather than eyeballing them. Kept the source hex in CSS comments so future edits can round-trip without re-deriving.
- The shadcn Nova preset shipped with a latent font bug: `next/font` bound Geist Sans to `--font-geist-sans`, but `globals.css` `@theme inline` had `--font-sans: var(--font-sans);` — a self-referential mapping. So Tailwind's `font-sans` utility was resolving to nothing and the page silently fell back to the browser default sans-serif rather than Geist. Fixed by renaming the next/font variables to `--font-sans` / `--font-mono` and registering `--font-serif` alongside, then verifying via `getComputedStyle` on synthetic `font-sans` / `font-mono` / `font-serif` nodes that all three resolve to the right family.
- Capturing the typing indicator in a screenshot during a real stream turned out to be impossible — `evaluateAccusation` and the streaming token cadence are both fast enough that the empty-content gate (`message.content === '' && index === total - 1 && isStreaming`) closes before any preview eval can sample it. Worked around by injecting synthetic `<span class="dot dot-N" />` markup into the DOM and reading `getComputedStyle` to confirm `animation-name: dot-pulse`, `1.4s infinite ease-in-out`, `0/150/300ms` delays, and `var(--muted-foreground)` color — same outcome, deterministic.

## Mistakes / things I'd do differently

- §2.1 of the spec described the accusation Submit loading as a "disabled spinner state," but §4.5 explicitly defers the spinner icon to W4. I caught the inconsistency only at Step 7 (during implementation) and kept §4.5 as binding. Lesson: when a spec has a high-level scope bullet *and* a detailed sub-section, reconcile them at draft time, not when writing the implementation. Annotated §2.1 in Step 8 to flag the resolution.
- I left the suspect-card on Briefing without a hover/transition state because there's only one suspect and no click target. The original §2.1 bullet listed "suspect cards" alongside tabs and picker rows, which implied parity. With three suspects in W3, this becomes an actual decision (interactive bio? decorative card?) — better to have answered it in this weekend's spec rather than punted to W3 planning.
